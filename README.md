**Today I Learned**

```
< git bash >

ls : 파일 리스트 목록
cd : 폴더 경로 이동
git init : 해당 폴더 git 에서 관리
ls -a : 관리자명 입력

이미 git 저장소인 폴더 내에 또 다른 git 저장소를 만들지 않는다.
절대로 홈 디렉토리에서 git init를 하지 않는다. 

git status : 파일의 현재 상태를 알려주는 명령어
touch : 텍스트 만들어 주는 명령어
cd .. : 앞 경로로 이동
rm -r : 폴더 삭제
rm -rf : 강제 삭제

(2) **git status**

- working directory와 staging area에 있는 파일의 현재 상태를 알려주는 명령어
- 어떤 작업을 시행하기 전에 수시로 status를 확인하면 좋다.

(3) **git add**

- working dierctory에 있는 파일을 staging area로 올리는 명령어

(4) **git commit**

- staging area에 올라온 파일의 변경 사항을 하나의 버전으로 저장하는 명령어

(5) **git log**

커밋의 내역(id,작성자,시간,메세지 등)을 조회할 수 있는 명령어
```
